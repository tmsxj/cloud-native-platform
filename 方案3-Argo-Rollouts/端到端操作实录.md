## 三、创建 Kustomize 文件结构

### 3.1 Base Rollout 模板

```bash
mkdir -p k8s/base k8s/overlays/{dev,staging,prod}
```

**k8s/base/rollout.yaml** — 核心变更：`kind: Deployment` → `kind: Rollout`

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Rollout
metadata:
  name: tomcat-app
  labels:
    app: tomcat-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: tomcat-app
  template:
    metadata:
      labels:
        app: tomcat-app
    spec:
      imagePullSecrets:
        - name: harbor-regcred
      containers:
        - name: tomcat-app
          image: 192.168.1.61/tomcat-demo/tomcat-app:latest   # ← 改用 latest（v1-init 已被 GC）
          imagePullPolicy: Always
          ports:
            - containerPort: 8080
              name: http
              protocol: TCP
          livenessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 40
            periodSeconds: 15
          readinessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 30
            periodSeconds: 10
          resources:
            requests:
              memory: "256Mi"
              cpu: "100m"
            limits:
              memory: "512Mi"
              cpu: "500m"
          env:
            - name: JAVA_OPTS
              value: "-Xms128m -Xmx384m"
```

### 3.2 三环境 Overlay 策略

#### DEV — 金丝雀 Canary

```yaml
# k8s/overlays/dev/kustomization.yaml 核心 patch
spec:
  replicas: 1
  strategy:
    canary:
      steps:
        - setWeight: 20
        - pause:
            duration: 60s
        - setWeight: 40
        - pause:
            duration: 60s
        - setWeight: 100
```

#### STAGING — 蓝绿 BlueGreen

```yaml
# k8s/overlays/staging/kustomization.yaml 核心 patch
spec:
  strategy:
    blueGreen:
      activeService: tomcat-app
      previewService: tomcat-app-preview    # ← 需额外创建
      previewReplicaCount: 1
      autoPromotionEnabled: false           # ← 手动推进
      scaleDownDelaySeconds: 120            # ← 旧版保留 2 分钟
```

#### PROD — 蓝绿 BlueGreen

```yaml
# k8s/overlays/prod/kustomization.yaml 核心 patch
spec:
  strategy:
    blueGreen:
      activeService: tomcat-app
      previewService: tomcat-app-preview
      previewReplicaCount: 1
      autoPromotionEnabled: false           # ← 手动推进
      scaleDownDelaySeconds: 600            # ← 旧版保留 10 分钟
  # + 高资源配额、严格探针
```

## 五、问题修复

### 5.1 ⚠️ BlueGreen InvalidSpec — kustomize commonLabels 与 CRD

**现象**：
```
Error: Rollout "tomcat-app" is invalid:
spec.strategy.blueGreen.activeService: Invalid value "tomcat-app":
Service tomcat-app has unmatch label env in rollout
```

**排查过程**：
1. `kubectl get svc tomcat-app -n tomcat-staging -o yaml | grep -A5 selector` → 有 `env: staging`
2. `kubectl get rollout tomcat-app -n tomcat-staging -o yaml | grep -A5 matchLabels` → **没有** `env: staging`

**根因**：kustomize `commonLabels` 只对标准 K8s 资源生效，Rollout 是 CRD，kustomize 无法识别其结构。因此 `env: staging` 标签被注入到 Service selector，但未注入到 Rollout 的 `spec.selector.matchLabels` 和 `spec.template.metadata.labels`。

**修复**：移除所有 overlay 的 `commonLabels`，改用显式 JSON patch：

```yaml
# 在每个 overlay kustomization.yaml 中：
patches:
  - target:
      kind: Rollout
      name: tomcat-app
    patch: |-
      - op: add
        path: /metadata/labels/env
        value: "staging"
      - op: add
        path: /spec/template/metadata/labels/env
        value: "staging"
      - op: add
        path: /spec/selector/matchLabels/env
        value: "staging"
  - target:
      kind: Service
      name: tomcat-app
    patch: |-
      - op: add
        path: /spec/selector/env
        value: "staging"
```

> **教训**：Rollout CRD 的标签管理**必须用显式 JSON patch**，`commonLabels` 不适用。

### 5.2 ⚠️ Secret 被占位符覆盖

**现象**：`kubectl apply -k` 后 Secret 内容变成 `REPLACE_WITH_BASE64`，导致 Pod 无法拉取镜像。

**根因**：kustomization.yaml 的 `resources` 中包含 `secret.yaml`，kustomize 会用文件内容覆盖集群中已存在的 Secret。

**修复**：从 `resources` 列表中移除 `secret.yaml`，Secret 已存在于集群中：

```yaml
# overlays/<env>/kustomization.yaml
resources:
  - ../../base
  - namespace.yaml
  # secret.yaml 已存在于集群，此处仅作参考模板（注释掉）
```

### 5.3 ⚠️ ImagePullBackOff — v1-init 已清理

**现象**：
```
Failed to pull image "192.168.1.61/tomcat-demo/tomcat-app:v1-init":
manifest for 192.168.1.61/tomcat-demo/tomcat-app:v1-init not found
```

**根因**：Harbor 的垃圾回收清理了旧 tag `v1-init`。

**修复**：

```bash
# 1. Base rollout.yaml 改用 latest tag
# image: 192.168.1.61/tomcat-demo/tomcat-app:latest

# 2. 重新 apply
kubectl apply -k k8s/overlays/dev
kubectl apply -k k8s/overlays/staging
kubectl apply -k k8s/overlays/prod
```

### 5.4 ⚠️ 旧 ReplicaSet 残留失败 Pod

**现象**：新 Rollout 已 Healthy，但仍有旧 ReplicaSet 的 Pod 反复 Crashing。

**根因**：金丝雀和蓝绿策略会保留旧 ReplicaSet 用于回滚。旧 RS 使用了已删除的镜像 tag，K8s 持续尝试重建失败 Pod。

**修复**：手动缩容旧 ReplicaSet：

```bash
# DEV 旧 RS
kubectl scale rs tomcat-app-599cc9b6d -n tomcat-dev --replicas=0

# PROD 旧 RS
kubectl scale rs tomcat-app-6758f45cf9 -n tomcat-prod --replicas=0

# STAGING 旧 RS 已为 0，无需操作
```
