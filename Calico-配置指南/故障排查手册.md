# Calico（CNI/网络策略方案） 故障排查手册

> 配套 `README.md` / `Calico-生产配置详解.md`，常见故障快速定位。

---

## 一、Pod（容器组） 网络不通（跨节点）

| 可能原因 | 排查 |
|:---|:---|
| 封装模式不匹配 | `kubectl get ippool -o yaml` 确认 `ipipMode/vxlanMode` 与底层网络一致 |
| MTU 不一致 | 两端 MTU 需相同；隧道场景减 20/50；`ip link show` 查 `cali*`/`tunl*` |
| BGP 未建立 | `calicoctl node status` 看 Peer 状态 |

## 二、Pod 无法获取 IP

| 可能原因 | 排查 |
|:---|:---|
| IPPool 耗尽 | `calicoctl ipam show` 看剩余；扩 CIDR 或减小 blockSize |
| nodeSelector 不匹配 | 确认 IPPool `nodeSelector` 覆盖该节点 |
| `calico-node` 未就绪 | `kubectl -n calico-system get pods` 看 CrashLoop / OOM |

## 三、NetworkPolicy 不生效

| 可能原因 | 排查 |
|:---|:---|
| 数据面为 iptables 但规则缺失 | `iptables-save | grep cali` 确认链存在 |
| BPF 模式未加载 | `calicoctl` 查 felix 状态；确认 `linuxDataplane: BPF` |
| 策略命名空间/标签错误 | 核对 `podSelector` 与 Pod 实际 labels |

## 四、MTU 相关偶发丢包

- 现象：大文件传输慢、TLS 握手偶败、TCP 重传高
- 定位：`ping -s 1472 <pod-ip>` 逐步减小包大小，找到分水岭
- 解决：统一两端 MTU（隧道场景 VXLAN 1450 / IPIP 1480）

## 五、Typha 故障（大规模）

| 现象 | 排查 |
|:---|:—|
| felix 频繁重载 | 查 Typha Pod 是否 OOM；`kubectl -n calico-system logs typha-xxx` |
| API server 压力高 | 确认 Typha 已启用且副本数足够 |

## 六、快速诊断命令

```bash
calicoctl get nodes -o wide
calicoctl ipam show
kubectl -n calico-system get pods
kubectl -n calico-system logs -l k8s-app=calico-node --tail=50
```
