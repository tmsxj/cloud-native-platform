# 达梦 DM8 部署记录

> 版本：2026-04-27 | 环境：t1 (Ubuntu) | 安装方式：命令行静默安装

## 一、环境准备

### 1.1 磁盘扩容（关键前置步骤）

部署前根分区仅 **10G**，使用率已达 **96%**，无法容纳 DM8 安装文件。

**排查过程：**

```bash
# 1. 查看磁盘使用
df -h /
# /dev/mapper/ubuntu--vg-ubuntu--lv  9.8G  9.4G  0.4G  96% /

# 2. 查看 LVM 结构
sudo fdisk -l /dev/sda
sudo pvs && sudo vgs && sudo lvs
# VG ubuntu-vg 有 8.22G 空闲空间
# LV ubuntu-lv 当前 10G
```

**扩容操作：**

```bash
# 在线扩容 LV + 文件系统（一步到位）
sudo lvextend -L +8G -r /dev/ubuntu-vg/ubuntu-lv
# 结果：10G → 18G，可用空间 8.0G
```

> **要点**：LVM 在线扩容不需要重启，`-r` 参数自动 resize 文件系统。

## 二、静默安装

### 2.1 挂载 ISO

```bash
sudo mkdir -p /mnt/dm8
sudo mount -o loop \
  /home/tmsxj/db-install/dm8_install/dm8_20260427_x86_CentOS7_64.iso \
  /mnt/dm8

ls /mnt/dm8/
# DMInstall.bin  DMInstall.pdf  ...
```

### 2.2 编写静默安装配置文件

创建 `auto_install.xml`：

```xml
<?xml version="1.0"?>
<DATABASE>
    <LANGUAGE>en</LANGUAGE>           <!-- 英文系统必须用 en -->
    <TIME_ZONE>+08:00</TIME_ZONE>
    <KEY></KEY>
    <INSTALL_TYPE>0</INSTALL_TYPE>    <!-- 0=典型安装 -->
    <INSTALL_PATH>/home/tmsxj/dm8/dmdbms/</INSTALL_PATH>
    <INIT_DB>N</INIT_DB>              <!-- 先不自动初始化，手动 init 更可控 -->
    <CREATE_DB_SERVICE>N</CREATE_DB_SERVICE>
    <STARTUP_DB_SERVICE>N</STARTUP_DB_SERVICE>
</DATABASE>
```

### 2.3 执行安装

```bash
# 设置语言环境
export LANG=en_US.UTF-8

# 静默安装（-q 参数 + XML 配置文件）
/mnt/dm8/DMInstall.bin -q /home/tmsxj/auto_install.xml
```

### 2.4 执行 root 脚本

```bash
# DM 安装后会生成 root 脚本，必须执行
sudo /home/tmsxj/dm8/dmdbms/script/root/root_installer.sh
```

## 三、手动初始化数据库

### 3.1 初始化实例

```bash
# 指定 PATH 为 /data（根分区空间有限）
/home/tmsxj/dm8/dmdbms/bin/dminit \
  PATH=/data/dm8/data \
  DB_NAME=DMDB \
  INSTANCE_NAME=DMINSTANCE \
  PORT_NUM=5236 \
  PAGE_SIZE=16 \
  LOG_SIZE=256 \
  CASE_SENSITIVE=Y \
  CHARSET=0 \
  SYSDBA_PWD=Dameng123 \
  SYSAUDITOR_PWD=Dameng123
```

**参数说明：**

| 参数 | 值 | 说明 |
|------|-----|------|
| PAGE_SIZE | 16 | 页大小 16KB |
| LOG_SIZE | 256 | redo 日志大小 256MB |
| CASE_SENSITIVE | Y | 大小写敏感（兼容 Oracle） |
| CHARSET | 0 | GB18030 字符集 |

### 3.2 注册系统服务

```bash
sudo /home/tmsxj/dm8/dmdbms/script/root/dm_service_installer.sh \
  -t dmserver -p DMDB -dm_ini /data/dm8/data/DMDB/dm.ini
```

### 3.3 启动与验证

```bash
# 启动服务
sudo systemctl start DmServiceDMDB

# 检查状态
sudo systemctl status DmServiceDMDB --no-pager

# 连接验证
/home/tmsxj/dm8/dmdbms/bin/disql SYSDBA/Dameng123@localhost:5236

SQL> SELECT 1 FROM DUAL;        -- 返回 1
SQL> SELECT name, status$ FROM V$DATABASE;
```

## 五、踩坑全记录

### 5.1 磁盘空间不足

**现象**：`df -h` 显示根分区 96% 满，DM8 安装需要约 2G+。

**解决**：LVM 在线扩容 `lvextend -L +8G -r`，一步完成无需重启。

### 5.2 静默安装语言不匹配 ⚠️ 关键坑

**现象**：
```
The installer has detected that your system is not a Chinese environment
This installation package only supports Chinese language
```

**原因**：XML 中 `LANGUAGE=zh`，但系统 `locale` 为 `en_US.UTF-8`。

**解决**：将 XML 中 `<LANGUAGE>zh</LANGUAGE>` 改为 `<LANGUAGE>en</LANGUAGE>`。

### 5.3 安装成功但数据库未初始化

**现象**：安装完成后 `/data/dm8/data/` 目录为空。

**原因**：`auto_install.xml` 中 `INIT_DB=Y` 自动初始化失败，且 root 脚本未执行。

**解决**：
1. 先执行 `root_installer.sh`
2. 手动 `dminit` 初始化
3. 手动 `dm_service_installer.sh` 注册服务

**经验总结**：手动初始化比自动初始化更可控，推荐 `INIT_DB=N`，分步操作。

### 5.4 dminit 参数错误

**现象**：
```
LOG_SIZE must between 256 and 4096
```

**原因**：DM8 的 LOG_SIZE 最小 256MB，填写 64 或 128 都会报错。

**解决**：`LOG_SIZE=256`（最小值）。

### 5.5 数据目录空间不足

**现象**：第一次 `dminit` 在根分区执行，文件生成到一半磁盘满了。

**解决**：将 `PATH` 指向 `/data/dm8/data`（20G 独立分区），清理根分区失败产物后重试。

### 5.6 PowerShell SSH 转义问题

**现象**：通过 SSH 远程执行时，单引号、双引号、`$` 符号被 PowerShell 截获或转义。

**解决**：
- 复杂脚本用 `scp` 传文件到远程再执行
- 或用 `cat > file << 'EOF'` heredoc（不加引号限制无法在 PowerShell 中正确传递）
- 最可靠的方式：PowerShell 本地写文件 → `scp` 上传 → SSH 远程执行

### 5.7 ISO 挂载后清理

**现象**：安装完成后 ISO 仍占用挂载点，根分区空间未释放。

**解决**：
```bash
sudo umount /mnt/dm8
sudo rm -rf /mnt/dm8
rm -rf /home/tmsxj/db-install/dm8_install/  # 清理安装源文件
```
