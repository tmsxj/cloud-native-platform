# 国产数据库部署 — 通用踩坑汇总

> 涵盖达梦 DM8 / 崖山 YashanDB 部署过程中的通用问题和应对策略。

## 一、系统环境类

| 坑 | 现象 | 排查方法 | 解决方案 |
|----|------|----------|----------|
| **磁盘空间不足** | 安装/初始化中途失败，`No space left on device` | `df -h` 查看各分区使用率 | LVM 扩容或清理无用文件，数据目录放到独立分区 |
| **内存不足** | 安装程序 OOM 被 kill | `free -h` 查看可用内存 | 关闭非必要服务，或调低数据库内存参数 |
| **系统语言不匹配** | 达梦静默安装报 "not a Chinese environment" | `echo $LANG` | XML 中 LANGUAGE 改为 en，或 `export LANG=zh_CN.UTF-8` |
| **内核参数未调优** | 数据库启动报 semaphore 或 shm 相关错误 | `sysctl -a | grep -E 'kernel.sem|kernel.shm'` | 参考官方文档调整 `/etc/sysctl.conf` |

## 二、安装部署类

| 坑 | 现象 | 排查方法 | 解决方案 |
|----|------|----------|----------|
| **静默安装 XML 格式错误** | 安装程序解析 XML 失败 | `xmllint` 或 `python3 -c "import xml.etree.ElementTree"` 校验 | 确保 XML 闭合标签正确、特殊字符转义 |
| **root 脚本未执行** | 服务注册失败、数据库无法启动 | `ls /home/tmsxj/dm8/dmdbms/script/root/` | 安装后立即执行 root 脚本 |
| **端口被占用** | `bind: address already in use` | `ss -tlnp | grep 端口号` | 换端口或停掉占用进程 |
| **环境变量未加载** | `yasboot/yasql` 或 `disql` 命令找不到 | `which yasboot` / `which disql` | 安装后 source 环境变量文件或写入 `~/.bashrc` |

## 三、数据库初始化类

| 坑 | 现象 | 排查方法 | 解决方案 |
|----|------|----------|----------|
| **参数值超出范围** | `LOG_SIZE must between 256 and 4096` 等 | 查阅官方文档参数取值范围 | 确认每个参数的最小/最大值 |
| **数据目录不在独立分区** | 根分区很快被数据文件写满 | `df -h` 对比数据目录所在分区 | 数据目录 PATH 指向 `/data` 或独立分区 |
| **字符集选错** | 中文乱码或排序异常 | `SELECT * FROM V$NLS_PARAMETERS` | CHARSET=0(GB18030) 适合政务场景；1=UTF-8 适合通用场景 |

## 四、服务管理类

| 坑 | 现象 | 排查方法 | 解决方案 |
|----|------|----------|----------|
| **systemd 服务未注册** | `systemctl start` 找不到服务 | `systemctl list-unit-files | grep -i dm` | 手动执行 `dm_service_installer.sh` 注册 |
| **启动后端口不通** | `telnet localhost 5236` 拒绝连接 | `ss -tlnp | grep 5236` + 查看错误日志 | 检查 `dm.ini` 端口配置、检查防火墙规则 |
| **sudo 权限问题** | `permission denied` 操作系统服务 | 确认当前用户在 sudo 组 | `echo 'password' | sudo -S command` 或 `sudo su -` |

## 五、远程操作类（PowerShell → SSH）

| 坑 | 现象 | 排查方法 | 解决方案 |
|----|------|----------|----------|
| **引号转义混乱** | 单引号、双引号、`$` 被截断或展开 | 本地对比目标文件内容差异 | 用 `scp` 传文件，避免直接在 SSH 命令中写复杂字符串 |
| **管道挂起** | SSH 命令执行后无输出不退出 | 进程列表看 SSH 是否挂起 | `ssh host "cmd" 2>&1` 或加超时 |
| **sudo 密码交互** | 非交互式环境下 sudo 卡住等待密码 | 命令不返回 | `echo 'pass' | sudo -S cmd` 管道传密码 |

## 六、核心经验总结

1. **数据目录放独立分区**：根分区爆满是最常见的生产事故，从一开始就规划好独立数据盘。
2. **手动初始化优于自动初始化**：静默安装 XML 中设置 `INIT_DB=N`，分步 init 更容易定位问题。
3. **安装→root 脚本→初始化→注册服务→启动**：严格按这个顺序，中间任何一步失败都能独立排查。
4. **远程操作优先传文件**：PowerShell 的引号转义是个大坑，复杂操作一律 `scp` + 远程执行。
5. **每次失败先看错误日志**：数据库安装失败不要盲试，`tail -100` 错误日志往往直接给出原因。
