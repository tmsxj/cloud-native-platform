# 术语表 / Glossary

> K8s + DevOps 常见缩写和概念速查。按首字母排列，方便快速查阅。

---

## K8s 核心概念

| 缩写/术语 | 全称 | 一句话解释 |
|----------|------|-----------|
| **CRD** | Custom Resource Definition | 自定义资源定义。让 K8s 认识新的资源类型（如 ArgoCD 的 Application、Rollout） |
| **HPA** | Horizontal Pod Autoscaler | 水平弹性伸缩。根据 CPU/内存指标自动增减 Pod 副本数 |
| **Ingress** | — | K8s 的资源类型，定义外部流量如何路由到集群内的 Service（域名→Service→Pod） |
| **NodePort** | — | Service 的一种类型，在每个节点上开放一个端口（30000-32767），外部可通过 `<节点IP>:<端口>` 访问 |
| **PVC** | PersistentVolumeClaim | 持久化存储声明。Pod 通过 PVC 申请存储空间，类似"我需要 5Gi 的硬盘" |
| **PV** | PersistentVolume | 持久化存储卷。集群管理员创建或 StorageClass 动态创建的物理存储 |
| **RBAC** | Role-Based Access Control | 基于角色的权限控制。控制谁（Subject）能对什么资源（Resource）做什么操作（Verb） |
| **SA** | ServiceAccount | 服务账户。Pod 运行时的身份标识，用于 Pod 访问 K8s API 时的认证 |
| **SC** | StorageClass | 存储类。定义存储的"类别"（NFS/SSD/local），配合 Provisioner 实现动态创建 PV |
| **StatefulSet** | — | 有状态应用控制器。每个 Pod 有固定编号 + 独立 PVC，适合数据库等有状态服务 |

## DevOps / CI/CD

| 缩写/术语 | 全称 | 一句话解释 |
|----------|------|-----------|
| **CI** | Continuous Integration | 持续集成。代码提交后自动构建、测试、打包，确保每次提交都是可部署的 |
| **CD** | Continuous Delivery / Deployment | 持续交付/部署。将通过 CI 验证的代码自动部署到目标环境 |
| **GitOps** | — | 以 Git 仓库作为声明式基础设施和应用的唯一真相源。所有变更通过 Git commit 驱动 |
| **Pipeline** | — | 流水线。将 CI/CD 过程拆成多个阶段（Stage），按顺序或并行执行 |
| **DIND** | Docker-in-Docker | 在容器内运行 Docker 守护进程。用于 K8s 集群中执行 `docker build` 命令 |
| **JWT** | JSON Web Token | 一种紧凑的 URL 安全令牌格式，用于认证和信息交换 |

## ArgoCD 相关

| 术语 | 一句话解释 |
|------|-----------|
| **Application** | ArgoCD 的核心 CRD。定义"哪个 Git 仓库的哪个目录 → 部署到哪个 K8s 集群的哪个命名空间" |
| **sync（同步）** | ArgoCD 将 Git 中 YAML 的期望状态应用到 K8s 集群的实际操作 |
| **auto-sync** | 自动同步模式。ArgoCD 检测到 Git 变更后自动执行 sync，无需人工操作 |
| **selfHeal** | 自愈模式。发现集群实际状态与 Git 声明不一致时自动修复（如有人手动改了 K8s） |
| **prune** | 修剪模式。Git 中删除的 YAML 文件，ArgoCD 自动从集群中清理对应的资源 |
| **outOfSync** | ArgoCD 检测到集群状态与 Git 声明存在差异的状态标识 |
| **Argo Rollouts** | Argo 系列中的渐进式发布工具。支持金丝雀(Canary)和蓝绿(BlueGreen)发布策略 |
| **Canary（金丝雀）** | 逐步将流量从旧版本切换到新版本（如 20%→40%→100%） |
| **BlueGreen（蓝绿）** | 新版本(绿)完全部署好后再一次性切换流量，旧版本(蓝)保留一段时间作为回滚备份 |

## 可观测性 (Observability)

| 缩写/术语 | 全称 | 一句话解释 |
|----------|------|-----------|
| **三支柱** | Metrics + Logs + Traces | 可观测性的三大数据来源。指标告诉你"有什么问题"、日志告诉你"为什么出问题"、链路告诉你"哪里出问题" |
| **TSDB** | Time Series Database | 时序数据库。Prometheus 的后端存储引擎，专门存随时间变化的数值数据（CPU/内存/QPS） |
| **OAP** | Observability Analysis Platform | SkyWalking 的后端分析平台。接收 agent 上报的 trace 数据，做聚合分析和拓扑计算 |
| **AlertManager** | — | Prometheus 生态的告警管理组件。负责告警的分组、抑制、静默，并通过多种渠道（邮件/Webhook/钉钉）发送通知 |
| **Promtail** | — | Loki 的日志采集 agent。类似 Prometheus 的 node-exporter，但采集的是日志而非指标 |
| **Loki** | — | Grafana 出品的轻量级日志聚合系统，查询语法与 Prometheus 的 PromQL 类似 |

## 镜像与安全

| 缩写/术语 | 一句话解释 |
|----------|-----------|
| **Harbor** | VMware 开源的私有 Docker 镜像仓库。支持镜像复制、漏洞扫描、RBAC 权限控制 |
| **Trivy** | Aqua Security 开源的安全扫描工具。支持容器镜像、Git 仓库、文件系统的漏洞扫描 |
| **Sealed Secrets** | Bitnami 开源的 K8s Secret 加密方案。将 Secret 加密后可以安全存入 Git，集群内自动解密 |

## 工具链

| 缩写/术语 | 一句话解释 |
|----------|-----------|
| **Kustomize** | K8s 原生的配置管理工具。通过 base + overlays 模式管理多环境差异，无需模板语言 |
| **Helm** | K8s 的包管理器。将一组 YAML 打包为 Chart，支持模板变量和依赖管理 |
| **envsubst** | 环境变量替换工具。将 YAML 中的 `${VAR}` 占位符替换为实际值。本项目用它实现参数化部署 |

## 网络相关

| 缩写/术语 | 一句话解释 |
|----------|-----------|
| **TLS** | Transport Layer Security | 传输层安全协议。HTTPS 的基础，加密客户端和服务器之间的通信 |
| **Ingress Controller** | Ingress 规则的实际执行者（如 nginx-ingress）。监听 K8s API 的 Ingress 变更，动态渲染 nginx 配置 |
| **cert-manager** | Jetstack 开源的证书管理工具。自动为 Ingress 签发和续期 TLS 证书（支持 Let's Encrypt） |
| **LoadBalancer** | Service 的一种类型。调用云厂商的 API 创建外部负载均衡器，每个 Service 独占一个公网 IP |

## 本项目中的一些习惯用语

| 术语 | 在本项目中的含义 |
|------|----------------|
| **env.sh** | 统一环境配置文件，所有部署脚本共享 |
| **deploy-helper.sh** | envsubst 模板渲染函数库，提供 `apply_template()` 等通用部署函数 |
| **方案 1 / P1** | Jenkins + ArgoCD 方案（Python Demo） |
| **方案 2 / P2** | GitLab CI + ArgoCD 方案（Java Tomcat Demo） |
| **方案 3 / P3** | Argo Rollouts 灰度发布方案（金丝雀 + 蓝绿） |
| **监控三支柱** | Metrics (Prometheus) + Logs (Loki) + Traces (SkyWalking) |
| **K8s 三基石** | HPA (弹性伸缩) + PV/PVC (持久存储) + Ingress (流量入口) |
| **实例 / target** | Prometheus 采集目标。一个 scrape target 通常对应一个 Pod 或 Service |
| **job** | Prometheus 中的采集任务。一个 job 对应一组同类型的 scrape targets |
