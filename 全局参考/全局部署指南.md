# 项目实战 - 全局部署指南

> 从零开始在新集群上拉起 CI/CD + 可观测性全链路

---

## 一、部署前准备

### 1.1 前置条件

| 组件 | 要求 | 说明 |
|------|------|------|
| K8s 集群 | 可用节点 ≥ 5 个 | 建议总内存 ≥ 32Gi |
| kubectl | 已配置集群访问 | `kubectl get nodes` 正常 |
| Ingress Controller | nginx-ingress 已部署 | `kubectl get pods -n ingress-nginx` |
| StorageClass | 默认 SC 就绪 | `kubectl get sc` |
| Harbor | 镜像仓库可访问 | 含 maven/tomcat/alpine/docker 等基础镜像 |
| Docker | 宿主机安装 Docker | 用于镜像推送和登录 |
| argocd CLI | `kubectl apply -f` 级也可 | 方便仓库注册 |

### 1.2 修改环境配置

```bash
cd 项目实战/
cp env.sh env.local.sh    # 创建本地配置，不上传 Git
vim env.local.sh           # 填入你的 Harbor IP/密码/Grafana 密码等
source env.local.sh        # 加载配置
```

**必改项：**
- `HARBOR_IP` — 你的 Harbor 地址
- `HARBOR_USER` / `HARBOR_PASS` — Harbor 凭据  
- `GRAFANA_PASS` — Grafana 管理员密码
- `STORAGE_CLASS` — 集群实际 SC 名称

## 三、分步骤部署

### 步骤 1: 创建命名空间 + 基础设施

```bash
source env.local.sh

# 创建所有命名空间
for ns in monitoring argocd jenkins git gitlab tomcat-demo tomcat-dev tomcat-staging tomcat-prod argo-rollouts; do
  kubectl create namespace $ns --dry-run=client -o yaml | kubectl apply -f -
done

# 创建镜像拉取凭证（在所有业务 NS）
for ns in monitoring tomcat-demo tomcat-dev tomcat-staging tomcat-prod; do
  kubectl create secret docker-registry harbor-regcred \
    -n $ns \
    --docker-server="$HARBOR_IP" \
    --docker-username="$HARBOR_USER" \
    --docker-password="$HARBOR_PASS" \
    --dry-run=client -o yaml | kubectl apply -f -
done
```

### 步骤 2: 部署 CI/CD（三选二）

#### 方案 1: Jenkins + ArgoCD

```bash
cd 项目实战/方案1-Jenkins-ArgoCD/配置文件/scripts/
source ../../../env.local.sh

```

组件清单：Git Server → ArgoCD → Jenkins → imagePullSecrets → ArgoCD Application

#### 方案 2: GitLab CI + ArgoCD

```bash

```

组件清单：GitLab → 镜像准备 → Runner → ArgoCD Application（3 环境）

### 步骤 3: 部署监控栈

```bash
cd 项目实战/可观测性/
source ../env.local.sh

# 如果有备份快照，用 restore 恢复
bash restore-monitoring.sh

# 如果没有，从 argocd/application.yaml 部署
kubectl apply -f argocd/application.yaml
```

监控组件清单（21 Pod）：
- **Metrics**: Prometheus + AlertManager + Node Exporter
- **Logs**: Loki + Promtail
- **Traces**: SkyWalking OAP + UI
- **Dashboard**: Grafana（含告警规则 + Dashboard JSON）

### 步骤 4: 三基石模块

```bash
# HPA 弹性伸缩
kubectl autoscale deployment tomcat-app -n tomcat-dev --cpu-percent=50 --min=1 --max=5

# PVC 持久化存储（静态 PV 方式）
kubectl apply -f pv-nfs.yaml -f pvc-tomcat-logs.yaml

# Ingress 路由
kubectl apply -f ingress-tomcat.yaml
```

## 五、故障恢复

### 5.1 监控缩容（省资源）

```bash
cd 项目实战/可观测性/
bash delete-monitoring.sh
```

### 5.2 监控恢复

```bash
cd 项目实战/可观测性/
bash restore-monitoring.sh backup-20260624-012453/
```

### 5.3 PVC 数据持久化

所有有状态组件均使用 PVC，删除 Pod 不会丢失数据：
- **Prometheus** TSDB: 50Gi
- **Grafana** SQLite: 10Gi
- **Loki**: 5Gi
- **Elasticsearch**: 10Gi
- **GitLab Data**: 30Gi
- **Jenkins**: 10Gi

## 七、常见问题

**Q: 内存不够跑全量监控？**  
A: 先跑方案 1/2/3 的基础 CI/CD（约 5Gi），监控栈按需启停。Prometheus 和 Grafana 可以独立运行不依赖 Loki/SkyWalking。

**Q: Harbor 镜像拉取失败？**  
A: 确认 Harbor 已 `docker login`，基础镜像已推送（maven/tomcat/alpine/gitlab-ce 等）。

**Q: ArgoCD 从 GitHub 安装失败（外网不通）？**  
A: 提前在有外网的机器下载 `install.yaml`，导入内网后 `kubectl apply -f`。

**Q: 想从备份恢复监控但集群没有备份目录？**  
A: `可观测性/backup-20260624-012453/` 目录内包含原始备份快照，可直接使用。
