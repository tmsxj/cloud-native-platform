# 项目实战仓库知识库重构指令集（项目实战专属 v1.0）

> **派生自**：`课程知识库批量重构_通用任务指令_v3.0.md`
> **定位**：把 v3.0 的「通用课程模板」收敛为**本项目实战仓库专用的可读性增强规范**。
> **核心差异**：v3.0 面向「课程资料 → 标准化课件」，本指令面向「真实集群测试环境沉淀的工程文件 → 更丰满易读的工程知识库」。**不套用三段式课程标题，以可读性增强为唯一目标。**

## 〇、适用范围与最终目标

- **适用仓库**：`项目实战`（GitHub `tmsxj/cloud-native-platform` ↔ Gitee `hlxb/cloud-native-platform`）
- **源材料性质**：离线 kubeadm 集群（master `192.168.1.51/52/53`、worker `192.168.1.54/55`、Harbor `192.168.1.61`、测试机 `192.168.1.71`）实测得到的部署记录、踩坑、验证闭环
- **最终目标（用户确认）**：原文基于测试环境、内容单薄不便阅读 → **补充内容、统一术语、补选型对比表，增强可读性**。不做课程化包装，不动工程实录价值。

## 一、环境信息层（本仓库真实参数）

## 二、重构原则（可读性增强，非课程化）

| 原则 | 本仓库落实方式 |
|------|----------------|
| 术语统一 | 首次出现标中文释义（如「Pod（容器组）」），全仓一致（对应 v3.0 R12） |
| 薄文件加深 | 少于 ~80 行的文件补真实命令、踩坑对照、验证输出 |
| 对比表补强 | 选型类内容补「为什么这么选」对比表（如 Calico vs Cilium） |
| 可执行命令 | 命令完整、含中文注释、标注预期输出（对应 R3 / R4） |
| 结构清晰 | 二级标题按内容自然分段，**不强制「理论/实操/面试」三段式** |
| 零占位 | 禁止 TODO / 待补充 / 方案A 等空洞词（对应 R1） |

## 五、加深操作 SOP（每次任务）

1. **定位目标**：按「类别 → 目录」挑薄文件（< ~80 行）或明显缺对比表的选型文。
2. **通读原文**：保留工程实录价值，不重写为课程体。
3. **补三类内容之一或多**：真实可执行的命令、踩坑对照表、选型对比表。
4. **术语扫描**：新词首次出现加中文释义，与全局术语表（如有）对齐。
5. **占位符自检**：正则扫 `TODO|待补充|方案[ABC]|\.\.\.|说明.*说明.*说明`，0 命中。
6. **提交**：本地 `git add -A && git commit`，**不自动推送**；用户授权后 `git push --force origin main`（GitHub + Gitee 双推，本地目录为准）。
