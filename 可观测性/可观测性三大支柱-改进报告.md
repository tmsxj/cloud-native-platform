### 🔴 新增 P0-X ✅ Loki 添加 retention（原无任何保留策略）

**问题**: Loki 配置中无 `retention_period`、无 `compactor`，日志会无限增长直到 3Gi local-path 写满 → Pod（容器组） CrashLoopBackOff

**改进**: 添加 `compactor` + `7天 retention`
```yaml
limits_config:
  retention_period: 168h
compactor:
  working_directory: /var/loki/compactor
  shared_store: filesystem
  compaction_interval: 10m
  retention_enabled: true
  retention_delete_delay: 2h
  retention_delete_worker_count: 150
```

### 🔴 新增 P0-X ✅ ES 添加 ILM 策略（原无索引清理）

**问题**: ES 单节点 5Gi，无索引生命周期策略 → 会无限增长

**改进**: 创建 `skywalking-cleanup` ILM 策略
```json
{
  "policy": {
    "phases": {
      "hot":  { "min_age": "0ms", "actions": { "rollover": { "max_size": "1gb", "max_age": "1d" }}},
      "delete": { "min_age": "14d", "actions": { "delete": {} }}
    }
  }
}
```

### 🟡 P1 ✅ Grafana 数据源预配

**问题**: Grafana Pod 重启后数据源丢失，需要手动重新配置

**改进**: 
1. 创建 `grafana-datasources` ConfigMap（配置字典），包含 Prometheus + Loki 数据源
2. 挂载到 `/etc/grafana/provisioning/datasources/`

```yaml
datasources:
  - name: Prometheus  # 默认数据源
    type: prometheus
    url: http://prometheus-server.monitoring:80
  - name: Loki
    type: loki
    url: http://loki.monitoring:3100
```

### 🟡 P1 ✅ Prometheus 告警规则补全

**原有规则** (5 条 - 仅节点级):
- NodeCPUHigh / NodeCPUCritical
- NodeMemoryHigh / NodeMemoryCritical
- NodeDiskPressure / NodeDiskUsageHigh

**新增规则** (8 条 - 补全 Pod/Deployment（部署，无状态工作负载）/PVC/组件级):

| 规则名 | 类型 | 严重级别 | 描述 |
|--------|------|----------|------|
| PodCrashLooping | 容器 | warning | Pod 15 分钟内重启 |
| PodNotReady | 容器 | warning | Pod 非 Running/Succeeded |
| PodOOMKilled | 容器 | critical | Pod 因内存不足被 Kill |
| DeploymentReplicasMismatch | 部署 | warning | 副本数未全部就绪 |
| PersistentVolumeUsage | 存储 | warning | PVC > 80% |
| PersistentVolumeCritical | 存储 | critical | PVC > 95% |
| KubeletDown | 节点 | critical | Kubelet 不可达 |
| KubeStateMetricsDown | 组件 | critical | kube-state-metrics 下线 |
